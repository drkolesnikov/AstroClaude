# Evaluation is qualitative-hermeneutic with rigorous provenance — no scoring metrics

Runs are judged by the analyst's recorded **reflection** (freeform: what landed,
what was generic or Barnum, what surprised) — not by numeric rubrics, scores, or
automated metrics. What *is* rigorous is **provenance**: every run auto-captures
its full config (selection, run mode, model, prompt + corpus versions, seed) so a
judgement is reproducible and attributable, and runs can be read side by side.

We reject scoring metrics because the research aim — archetypal *yield* — is
hermeneutic by definition; a number would import false precision and quietly
redefine "good" as "scoreable." Pairwise config comparison (same chart, two
configs, which surfaced more) is the planned next step; metric scoring is a
deliberate non-goal, not a missing feature.
