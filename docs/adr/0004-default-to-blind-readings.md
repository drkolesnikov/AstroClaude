# Default to blind readings; both run modes supported

A run reads either **blind** (birth data only) or **contextualized** (against the
native's known biography), as a per-run knob — and the default is **blind**. We
default to *not* using biography we may have, against the obvious instinct to use
everything, because the sandbox is judged hermeneutically (by the analyst's felt
sense of truth) and that judgement is acutely vulnerable to the **Barnum
effect**: a reading feels true when it merely echoes facts already fed in, or
stays vague enough to fit anyone. Blind mode is the only honest test that the
instrument surfaces something *non-obvious*; contextualized mode is reserved for
when clinical resonance — not validation of the method — is the goal.

## Consequences

- Every agent prompt and the evaluation protocol must distinguish the two modes.
- The anima/animus charter must function without gender (it is read fluidly
  regardless), so the instrument never *requires* biographical input.
- A reading that convinces only in contextualized mode is suspect, not validated.
